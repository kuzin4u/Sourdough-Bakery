# Пекарня на закваске — полное описание и инструкция по установке

Версия от 26.06.2026. Три компонента одного проекта — браузерный виджет и Telegram-бот
(две равноценные версии: Node.js и Python). Можно использовать любой компонент по
отдельности или все вместе — они синхронизируются между собой по номеру телефона
покупателя и общей Google Таблице.

```
bakery-bot-full/
├── README.md                          — этот документ
├── website/sourdough-shop.html        — браузерный виджет (каталог + корзина + AI-чат)
├── telegram-bot-node/                 — Telegram-бот на Node.js
├── telegram-bot-python/               — тот же бот на Python (полный аналог по функциям)
└── docs/FAQ_unified.md                — сводная база знаний о продукте + разбор источников
```

---

## 0. Что нового в этой версии

- **Модель Claude для AI-консультанта теперь настраивается, а не зашита в код.**
  По умолчанию — Haiku 4.5 (дешевле, для FAQ-вопросов хватает с запасом). При желании
  переключается на Sonnet 4.6 (более развёрнутые ответы, дороже ~3 раза) — без правки
  кода, через переменную окружения (боты) или поле в настройках (виджет).
- Полный перенос всех функций Telegram-бота с Node на Python (статусы заказа, стоп-лист,
  предзаказ на день выпечки, подписки, промокоды/лояльность, онлайн-оплата ЮKassa,
  запрос отзыва, админ-команды) — теперь обе версии равноценны.
- Единая база знаний FAQ, сведённая из двух источников пользователя, с явными
  правками неточных формулировок — встроена во все три компонента.
- Браузерный виджет учитывает стоп-лист бота (товар, помеченный как недоступный,
  показывается серым без кнопок и не предлагается AI-консультантом).

## 1. Что умеет каждый компонент

### 1.1 Браузерный виджет (`website/sourdough-shop.html`)

Самостоятельный мини-магазин, открывается как обычная HTML-страница (можно вставить
на сайт как iframe). Для базовой работы сервер не требуется.

| Функция | Описание |
|---|---|
| Каталог | 4 вида хлеба на закваске: вес, цена, описание |
| Корзина и чек-аут | количество, район доставки с расчётом стоимости, имя/телефон/адрес, способ оплаты |
| AI-консультант (чат) | отвечает на вопросы о составе/хранении/пользе по общей базе знаний; не принимает заказ напрямую — направляет в "Меню" |
| Выбор модели Claude | в настройках (⚙) — Haiku 4.5 по умолчанию или Sonnet 4.6 |
| Журнал заказов | постоянное хранилище артефакта, видно через настройки даже без интеграций |
| Уведомление в Telegram | опционально — заказ с сайта присылается личным сообщением бота |
| Запись в Google Таблицу | опционально — тот же заказ дублируется строкой в таблице |
| Единая корзина с Telegram | по номеру телефона — выбранное на сайте видно в боте и наоборот |
| Учёт стоп-листа | товары "нет в наличии" — серым, без кнопок, не предлагаются в чате |

### 1.2 Telegram-бот (Node и Python — функции идентичны)

| Функция | Команда/механика |
|---|---|
| Каталог и заказ | `/menu`, `/cart` — инлайн-кнопки, многошаговый чек-аут |
| Общая корзина с сайтом | привязка по телефону при первом `/menu` |
| AI-консультант (FAQ) | свободные вопросы вне сценария заказа, модель настраивается `CLAUDE_MODEL` |
| Статусы заказа | Принят → Готовится → Готов/в пути → Доставлен; клиенту уведомление автоматически |
| Стоп-лист | `/stop <id>`, `/instock <id>` (админ) — товар исчезает из каталога и ответов AI |
| Предзаказ на день выпечки | при оформлении — 3 ближайшие даты (настраивается `BAKE_WEEKDAYS`) |
| Подписка | `/subscribe` — фиксирует корзину + день недели; раз в 30 мин бот сам подкладывает набор и напоминает |
| Промокоды и лояльность | `/addpromo КОД percent 10` (админ); автоскидка 10% на каждый 5-й заказ |
| Онлайн-оплата (ЮKassa) | реальная интеграция, включается переменными окружения |
| Запрос отзыва | через `REVIEW_DELAY_HOURS` после заказа — оценка 1–5 звёзд админу |
| Админ-команды | `/today_orders`, `/stats` — без захода в Sheets |
| Общая Google Таблица | один webhook-URL для заказов из обоих каналов, колонка "Канал" |

Полное описание каждой механики и ограничений — в README внутри `telegram-bot-node/`
и `telegram-bot-python/`.

### 1.3 База знаний (`docs/FAQ_unified.md`)

Сведена из двух исходных FAQ-документов со честным разбором: что подтверждено фактами,
что смягчено как медицински неточное, и что отдельно помечено как непроверенная
бизнес-политика (возврат, межгородская доставка, продажа закваски) — эти пункты **не**
включены в ответы AI без подтверждения. База встроена в системные промпты всех трёх
компонентов и пересобирается на каждый запрос с учётом стоп-листа.

## 2. Архитектура

```
Покупатель в браузере          Покупатель в Telegram
        │                              │
        ▼                              ▼
 website/sourdough-shop.html    telegram-bot-node (или python)
        │                              │
        └──────────► общая корзина по номеру телефона ◄──────────┘
                      (через HTTP API бота)
                              │
                              ▼
                    единая Google Таблица заказов
                    (один и тот же Webhook URL везде)
                              │
                              ▼
                    уведомление вам в Telegram
```

Сайт и бот — независимые каналы продаж. Бот должен быть развёрнут и работать, чтобы
сайт мог с ним синхронизироваться — без этого сайт работает автономно, просто без
общей корзины и стоп-листа.

## 3. Установка и настройка — пошагово

### 3.1 Браузерный виджет — без сервера

1. Откройте `website/sourdough-shop.html` как файл — работает сразу.
2. AI-чат работает из коробки, ничего дополнительно настраивать не нужно.
3. Чтобы включить общую корзину/стоп-лист/уведомления — разверните бота (раздел 3.2),
   затем откройте настройки виджета (⚙) и впишите:
   - **адрес сервера бота** (выданный Render после деплоя)
   - **Telegram Bot Token + Chat ID** (если нужны уведомления о заказах с сайта)
   - **Google Sheets Webhook URL** (если нужна единая таблица заказов)
   - **модель Claude** — Haiku 4.5 оставить по умолчанию, или выбрать Sonnet 4.6

### 3.2 Telegram-бот — выбираете Node ИЛИ Python (функции одинаковые)

**Шаг 1 — три ключа, без них бот не запустится:**

| Что нужно | Где взять |
|---|---|
| `TELEGRAM_BOT_TOKEN` | [@BotFather](https://t.me/BotFather) в Telegram → `/newbot` |
| `ANTHROPIC_API_KEY` | [console.anthropic.com](https://console.anthropic.com/settings/keys) — отдельный аккаунт от claude.ai, нужна привязанная карта (Settings → Billing) |
| `ADMIN_CHAT_ID` | напишите [@userinfobot](https://t.me/userinfobot) — покажет ваш chat_id |

**Шаг 2 — локальный запуск (по желанию, можно сразу деплоить на Render):**

Node:
```bash
cd telegram-bot-node
npm install
cp .env.example .env        # заполните три ключа выше
npm start
```

Python:
```bash
cd telegram-bot-python
python3 -m venv venv && source venv/bin/activate
pip install -r requirements.txt
cp .env.example .env        # заполните три ключа выше
python main.py
```

Откройте бота в Telegram → `/start` → `/menu` — должен спросить телефон и показать каталог.

**Шаг 3 — какую модель Claude использовать:**

В `.env` есть переменная `CLAUDE_MODEL` (по умолчанию `claude-haiku-4-5-20251001`).
Менять не обязательно — Haiku 4.5 справляется с FAQ-вопросами с запасом и дешевле
примерно в 3 раза, чем Sonnet 4.6. Если ответы консультанта кажутся слишком сухими —
поставьте `claude-sonnet-4-6` и перезапустите бота, никаких других правок не требуется.

**Шаг 4 — деплой на Render (публичный адрес, нужен для связки с сайтом):**

Бот раздаёт HTTP API (корзина, стоп-лист), поэтому деплоится **только как Web Service**
(не Background Worker — у того нет публичного адреса).

1. Залейте папку (`telegram-bot-node/` или `telegram-bot-python/`) в репозиторий на
   GitHub. Файл `.env` в репозиторий не попадёт — он в `.gitignore`; это блокирует
   только Git, на работу самого бота локально не влияет.
2. На [render.com](https://render.com): **New → Blueprint** → подключите репозиторий —
   подхватит `render.yaml` и создаст Web Service автоматически. Render не ищет файлы
   по именам "по конвенции" — он просто выполняет команду из `render.yaml`
   (`startCommand`), поэтому переименовывать `main.py`/`index.js` не нужно и незачем.
3. План: **Starter (~$7/мес)** — без "засыпания"; **Free** — бесплатно, но "засыпает"
   после 15 мин без запросов (задержка ~1 мин на первый запрос после паузы).
4. В Environment добавьте переменные из `.env.example` — на Render они вводятся прямо
   в дашборде (Settings → Environment), а не через файл `.env`.
5. Deploy. Откройте `<выданный адрес>/api/health` — должно вернуться `{"status":"ok"}`.
6. Этот адрес вставьте в настройки браузерного виджета (поле "адрес сервера бота").

**Шаг 5 — проверка, что всё связалось:**
- `/start` → `/menu` в Telegram — спрашивает телефон, показывает каталог.
- На сайте впишите тот же адрес бота и тот же телефон в настройках (⚙).
- Добавьте товар в Telegram, обновите вкладку "Меню" на сайте — товар должен появиться
  в корзине сайта. Это подтверждает, что общая корзина работает.

### 3.3 Google Таблица заказов (опционально, общая для сайта и бота)

1. Откройте `sheets-apps-script.gs` (одинаковый файл в обеих папках бота) — инструкция
   в комментариях внутри файла.
2. Google Таблица → Расширения → Apps Script → вставьте код из файла.
3. Разверните как веб-приложение, скопируйте выданный URL.
4. Вставьте этот URL в три места: `.env` бота (`SHEETS_WEBHOOK_URL`), настройки сайта,
   и в `.env` второй версии бота, если используете обе параллельно.

### 3.4 Онлайн-оплата ЮKassa (опционально)

1. Зарегистрируйте магазин на [yookassa.ru](https://yookassa.ru), возьмите `shopId` и
   секретный ключ из личного кабинета.
2. Впишите в `.env` бота: `YOOKASSA_SHOP_ID`, `YOOKASSA_SECRET_KEY`.
3. В личном кабинете ЮKassa укажите webhook-адрес: `<адрес бота>/api/yookassa-webhook`.
4. Без этих переменных кнопка "Картой онлайн" честно говорит, что способ временно
   недоступен — остальной функционал работает как обычно.

## 4. Полный список переменных окружения (боты)

| Переменная | Обязательна? | Назначение |
|---|---|---|
| `TELEGRAM_BOT_TOKEN` | да | токен от @BotFather |
| `ANTHROPIC_API_KEY` | для FAQ-консультанта | ключ Anthropic |
| `CLAUDE_MODEL` | нет (умолч. Haiku 4.5) | `claude-haiku-4-5-20251001` или `claude-sonnet-4-6` |
| `ADMIN_CHAT_ID` | для уведомлений/админ-команд | ваш личный chat_id |
| `SHEETS_WEBHOOK_URL` | нет | URL Apps Script для единой таблицы заказов |
| `WEBHOOK_URL` (только Node) | нет | способ связи с Telegram (webhook вместо polling) |
| `BAKE_WEEKDAYS` | нет (умолч. 2,4,6) | дни недели выпечки для предзаказа |
| `REVIEW_DELAY_HOURS` | нет (умолч. 4) | через сколько часов просить отзыв |
| `YOOKASSA_SHOP_ID`, `YOOKASSA_SECRET_KEY` | нет | для онлайн-оплаты |
| `RETURN_URL` | нет | куда вернуть клиента после оплаты |
| `PORT` | нет (Render подставит сам) | порт HTTP-сервера |

## 6. Известные ограничения (честно, до продакшен-нагрузки)

- Все данные (заказы, корзины, подписки, промокоды) — в простых JSON-файлах. На
  бесплатном плане Render файловая система не персистентна между деплоями — данные
  обнулятся при каждом новом деплое кода (не при обычном перезапуске). Для серьёзной
  эксплуатации — платный план с диском или переезд на Postgres/Redis.
- Webhook ЮKassa не проверяет подпись запроса — для продакшена добавить проверку по
  документации ЮKassa.
- Состояние диалогов в Telegram — в памяти процесса, сбрасывается при перезапуске.
- Бизнес-политики из `FAQ_unified.md` (раздел 6: возврат, межгородская доставка, продажа
  закваски) не подтверждены и не включены в ответы AI — бот скажет "уточню у пекаря".
