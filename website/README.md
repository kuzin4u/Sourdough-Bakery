# Сайт пекарни — деплой на Render (Static Site)

Браузерный виджет (`sourdough-shop.html`) разворачивается на Render как отдельный
статический сайт, со своим доменом, независимо от Telegram-бота. При сборке
автоматически подставляется адрес сервера бота — открывшему сайт человеку не нужно
ничего вписывать вручную в настройках (⚙), поле уже будет заполнено.

## Как это устроено

- `sourdough-shop.html` — сам виджет, с плейсхолдером `__BOT_API_URL__` в одной строке кода
- `build.sh` — выполняется на этапе сборки Render, подставляет реальный адрес бота
  из переменной окружения `BOT_API_URL` вместо плейсхолдера, и публикует файл также
  как `index.html` (чтобы сайт открывался по корню домена)
- `render.yaml` — Blueprint-конфигурация для автодеплоя

Если переменная `BOT_API_URL` не задана — сайт всё равно соберётся и будет работать,
просто поле "Адрес сервера бота" в настройках останется пустым, и его нужно будет
вписать вручную, как до этого изменения.

## Деплой — пошагово

### Шаг 1 — сначала задеплойте бота (если ещё не сделали)

Сайту нужен публичный адрес бота — разверните `telegram-bot-node/` или
`telegram-bot-python/` по их собственным README, получите адрес вида
`https://sourdough-bakery-81ge.onrender.com`.

### Шаг 2 — репозиторий

Залейте папку `website/` (содержимое этой папки, не саму папку) в репозиторий на
GitHub — отдельный от репозитория бота, либо как отдельную папку в одном
монорепозитории (тогда при создании сервиса на Render укажите Root Directory =
`website`).

### Шаг 3 — создание сервиса

На [render.com](https://render.com): **New → Blueprint** → подключите репозиторий —
Render подхватит `render.yaml` и создаст Static Site автоматически.

Либо вручную: **New → Static Site** → выберите репозиторий →
- Build Command: `bash build.sh`
- Publish Directory: `.` (текущая папка)

### Шаг 4 — переменная окружения

В Environment сервиса добавьте:
```
BOT_API_URL = https://sourdough-bakery-81ge.onrender.com
```
(адрес вашего бота из Шага 1, без слэша на конце)

### Шаг 5 — деплой и проверка

Нажмите **Deploy**. Render выдаст адрес вида `https://sourdough-bakery-website.onrender.com`.
Откройте его — должен открыться виджет, и в настройках (⚙) поле "Адрес сервера бота"
уже должно быть заполнено автоматически.

## Обновление в будущем

Если меняете `sourdough-shop.html` (новые функции, правки) — просто залейте новую
версию файла в тот же репозиторий, Render пересоберёт сайт автоматически при пуше
(если включён Auto-Deploy) и снова подставит актуальный `BOT_API_URL`.

Если меняется сам адрес бота (например, переехали на другой план/сервис на Render) —
обновите значение переменной `BOT_API_URL` в Environment этого сайта и нажмите
**Manual Deploy → Deploy latest commit**, чтобы пересборка подхватила новое значение
(простое сохранение переменной без передеплоя не пересоберёт уже опубликованный HTML).

## Важно

- Демо-режим (превью внутри Claude.ai) и этот production-деплой используют один и
  тот же файл `sourdough-shop.html` — никакой отдельной "облегчённой" версии нет.
- Поле "Адрес сервера бота" в настройках (⚙) всё ещё можно отредактировать вручную
  после открытия сайта — сохранённое пользователем значение имеет приоритет над тем,
  что подставлено при сборке (на случай, если нужно временно указать другой адрес,
  например для теста).
